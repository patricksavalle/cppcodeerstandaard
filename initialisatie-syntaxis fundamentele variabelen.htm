Regel: initialisatie-syntaxis fundamentele types

Initialiseer variabelen van een fundamenteel type met toekenningssyntaxis, niet met een constructor-syntaxis.

Het voorbeeld geeft de geprefereerde syntaxis van initialisatie: 

// niet zo!
{
  int i( 0 );
  double d( 0.0 );
  char c( '\0' );
}

// maar zo...
{
  int i = 0;
  double d = 0.0;
  char c = '\0';
}

Sommige compilers zullen de eerste variant niet accepteren.
