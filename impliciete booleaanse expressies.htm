Regel: impliciete Booleaanse expressies

Schrijf elke impliciet aanwezige Booleaanse expressie altijd volledig uit.

Bijvoorbeeld, elk van de volgende statements is syntactisch correct:

  {
    // syntactisch correcte constructies, semantisch niet echt mooi! 

    int a = 0;
    void* b = 0;

    // bla bla 

    if (a)
    {
      // bla bla 
    }

    if (b)
    {
      // bla bla 
    }

    if (!a)
    {
      // bla bla 
    }

    if (!b)
    {
      // bla bla 
    }
  }

maar duidelijker is om ze als volgt te schrijven, zonder impliciete typecasts: 

  {
    // syntactisch én semantische correct constructies!

    int a = 0;
    void* b = 0;
  
    // bla bla 

    if (a == 0)
    {
      // bla bla 
    }

    if (b == 0)
    {
      // bla bla 
    }

    if (a != 0)
    {
      // bla bla 
    }

    if (b != 0)
    {
      // bla bla 
    }
  }

Een probleem dat zich kan voordoen bij eerder gegeven Booleaanse vergelijkingen is een eenvoudige verschrijving. In het onderstaande voorbeeld is deze gegeven. De code is syntactisch correct maar zal niet altijd even goed werken: 

  {
    if (a = 0)
    {
      // bla bla 
    }
  }

Het probleem is dat er een =-teken is vergeten, de expressie is nu altijd false én de variabele krijgt een waarde toegekend. Schrijf daarom altijd de constanten in de expressies links! Als er dan een =-teken wordt vergeten, ontstaat er een syntaxisfout. Het onderstaande voorbeeld is wellicht beter (hoewel minder duidelijk): 

  {
    // syntactisch én semantische correct constructies! 

    int a = 0;
    void* b = 0;
  
    // bla bla 

    if (0 == a)
    {
      // bla bla 
    }

    if (0 == b)
    {
      // bla bla 
    }

    if (0 != a)
    {
      // bla bla 
    }

    if (0 != b)
    {
      // bla bla 
    }
  }
