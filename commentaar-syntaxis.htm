Codeerstandaard voor C++

Regel: commentaar-syntaxis

Gebruik de // voor commentaar en gebruik geen geneste commentaren.

Geneste commentaren worden niet door alle compilers begrepen en zijn onduidelijk. Genest commentaar hoort ook niet thuis in productiecode. Soms moeten er echter tijdelijk grote blokken code worden uitgeschakeld, bijvoorbeeld voor testdoeleinden, of omdat de code nog niet gereed is. De kans bestaat dat deze code reeds is voorzien van commentaar. Het omsluiten van het blok code door commentaarhaken (/* */) kan dan ongewenste effecten geven; bijvoorbeeld omdat de compiler geen genest commentaar toestaat. Een goede manier om een blok code uit te schakelen is het gebruiken van onderstaande preprocessor-constructie: 

  // even tijdelijk wat code uitschakelen 
  
  #pragma message ("code disabled")
  #if 0

  // bla bla 

  #endif

Deze constructie werkt namelijk altijd goed, ook wanneer er in het uit te schakelen stuk code andere preprocessor-instructies voorkomen.
