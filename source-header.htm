Codeerstandaard voor C++

Regel: source-header

Geef elk bronbestand een standaard header.

Om een module snel te kunnen identificeren krijgt elke module een vaste header.

De volgende header volgens SCCS-conventie is geschikt voor gebruik met bijvoorbeeld het TrueChange RCS. 

  // ----------------------------------------------------------------
  // (C)1999 Lampje bv. All rights reserved        
  // ----------------------------------------------------------------
  // Project   : %R%
  // Module    : %M%
  // ----------------------------------------------------------------
  // Change-set: %L%:%B%:%S%
  // ----------------------------------------------------------------
  // Doel/beschrijving:
  //
  //   
  //
  // ----------------------------------------------------------------

Het RCS zal de metasymbolen (%R%, %L% enzovoort) expanderen tot leesbare tekst met informatie over de module en zijn revisiegeschiedenis.

De volgende is geschikt voor gebruik met het Microsoft VSS of Intersolv PVCS RCS: 

  // ----------------------------------------------------------------
  // (C)1999 Lampje bv. All rights reserved        
  // ---------------------------------------------------------------- 
  // $Author::                                                      $ 
  // $Logfile::                                                     $ 
  // $Date::                                                        $ 
  //
  // $Log:                $
  // 
  // $NoKeywords$ 
  // ----------------------------------------------------------------

De speciale door dollartekens gemarkeerde keywords worden door het versiecontrolesysteem geëxpandeerd. In de onderstaande tabel staan de door Microsoft VSS en Intersolv PVCS herkende keywords:

Keyword Beschrijving
$Archive: $ Visual SourceSafe archieflocatie
$Author: $ gebruiker die bestand het laatst wijzigde
$Date: $ datum en tijd van de oplevering laatste revisie
$Header: $ logfile, revisie, datum, auteur
$History: $ revisiegeschiedenis, VSS formaat
$Log: $ revisiegeschiedenis, RCS format
$Logfile: $ zelfde als $Archive: $
$Modtime: $ datum en tijd van laatste modificatie
$Revision: $ revisienummer
$Workfile: $ filenaam
$NoKeywords: $ geen keyword-expansion voor regels die volgen

Een voorbeeld van een geëxpandeerde header is het volgende codefragment: 

  // ----------------------------------------------------------------
  // (C)1999 Lampje bv. All rights reserved        
  // ---------------------------------------------------------------- 
  // $Author:: Savap                                                $ 
  // $Logfile:: /XLIBS/SEH/SRC/SEH.C                                $ 
  // $Date:: 3/26/97 4:38p                                          $ 
  //
  // $Log: /XLIBS/SEH/SRC/SEH.C $
  // 
  // 3     3/18/97 11:43a Savap
  // Helaas, helaas, een extra globale variabele was nodig. 
  // Zie revise 4 SEH.H.
  // 
  // 2     3/17/97 3:47p Hobbg
  // Exception stack corrected & enhanced: complete stack-trace since
  // THROW() which caused it.
  // 
  // 1     3/13/97 12:55p Savap
  // Initiële revisie. Foutloos compileerbaar.
  //
  // $NoKeywords$
  // ----------------------------------------------------------------
